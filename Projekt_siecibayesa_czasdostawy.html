Czas_Dostawy_Projekt

Filip Kosiorowski

2025-06-12

W ramach projektu zbudowano i przeanalizowano sieć bayesowską opisującą zależności pomiędzy czynnikami wpływającymi na czas dostawy w usługach logistycznych. Wykorzystano dane dotyczące m.in. warunków pogodowych, dystansu, poziomu ruchu drogowego, typu pojazdu i doświadczenia kuriera. Dane zostały uprzednio poddane faktoryzacji i dyskretyzacji, a następnie zastosowano metody heurystyczne (Hill-Climbing) do uczenia struktury sieci z zastosowaniem blacklisty i whitelisty.

# Wczytanie danych oraz wybór istotnych cech
# W ramach analizy przeprowadzono budowę oraz ocenę modeli sieci bayesowskich na podstawie danych dotyczących czasu dostaw.
# Celem analizy było zbadanie zależności pomiędzy cechami związanymi z dostawą a jej czasem.

dane <- read_csv("Food_Delivery_Times.csv", show_col_types = FALSE)

# Wybór istotnych cech do analizy
cechy <- c("Distance_km", "Weather", "Traffic_Level", "Time_of_Day", 
           "Vehicle_Type", "Courier_Experience_yrs", "Delivery_Time_min")
dane <- dane %>% select(all_of(cechy)) %>% na.omit()

# Faktoryzacja i dyskretyzacja danych
# Wybrane cechy poddano dyskretyzacji, a następnie przekształcono je na zmienne kategoryczne
# Każdą zmienną liczbową przekształcamy na kategorie dyskretne, zmienne binarne zamieniamy na faktory

dane <- dane %>%
  mutate(
    Weather = factor(Weather),
    Traffic_Level = factor(Traffic_Level),
    Time_of_Day = factor(Time_of_Day),
    Vehicle_Type = factor(Vehicle_Type),
    Doświadczenie = cut(Courier_Experience_yrs, breaks = c(-Inf,1,3,Inf),
                        labels = c("niskie", "średnie", "wysokie")),
    Czas_Dostawy = cut(Delivery_Time_min, breaks = c(-Inf,20,40,Inf),
                      labels = c("krótki", "średni", "długi")),
    Dystans = cut(Distance_km, breaks = c(-Inf,2,5,Inf),
                 labels = c("blisko", "średnio", "daleko"))
  ) %>%
  select(Weather, Traffic_Level, Time_of_Day, Vehicle_Type,
         Doświadczenie, Czas_Dostawy, Dystans) %>%
  mutate(across(everything(), as.factor)) %>%
  droplevels() %>%
  na.omit()

dane_final <- as.data.frame(dane)
# Uczenie struktury sieci bayesowskiej z blacklistą
# Wymuszamy, aby zmienna Czas_Dostawy nie była rodzicem żadnej innej zmiennej
# Dzięki temu analizujemy wpływ innych cech na czas dostawy

nodes <- names(dane_final)

  from = "Czas_Dostawy",

  stringsAsFactors = FALSE
)

# Wymuszenie krawędzi przychodzących do Czas_Dostawy — whitelist
# Zastosowano algorytm uczenia struktury z listą dozwolonych krawędzi, które wskazują na Czas_Dostawy jako zmienną zależną

nodes <- names(dane_final)
other_nodes <- setdiff(nodes, "Czas_Dostawy")

whitelist <- data.frame(
  from = other_nodes,
  to = rep("Czas_Dostawy", length(other_nodes)),
  stringsAsFactors = FALSE
)

blacklist <- data.frame(
  from = "Czas_Dostawy",
  to = other_nodes,
  stringsAsFactors = FALSE
)

struktura <- hc(dane_final, whitelist = whitelist, blacklist = blacklist)
graphviz.plot(struktura, main = "Sieć Bayesa z wymuszonymi krawędziami do Czas_Dostawy")

Wnioski

Z analizy wynika, że:

  • Czas dostawy w analizowanej próbce jest najczęściej długi (75%), znacznie rzadziej średni (22%) lub krótki (2%).

  • Dystans w większości przypadków jest daleki (91%), co może częściowo tłumaczyć wydłużony czas dostawy.

  • Najczęściej wykorzystywanym środkiem transportu jest rower (51%), rzadziej skuter (29%) i samochód (19%).

  • Najwięcej kurierów ma wysokie doświadczenie (ponad 61%), co sugeruje dominację zawodowych dostawców w próbie.

  • Ruch drogowy najczęściej jest klasyfikowany jako średni (40%) lub niski (39%).

  • Najwięcej dostaw przypada na porę poranną i wieczorną.

Z użyciem pakietu gRain przeprowadzono wnioskowanie probabilistyczne, które pozwoliło na przewidywanie zmiennych ukrytych przy zadanych wartościach innych. Przykładowe wyniki:

  • Jeżeli wiemy, że czas dostawy był długi, to z prawdopodobieństwem 91% dostawa była na daleki dystans.

  • Jeśli dystans był daleki, to największe prawdopodobieństwo (ok. 51%) przypisane jest wykorzystaniu roweru jako środka transportu.

  • Dla sytuacji: duży ruch i popołudniowa pora — najbardziej prawdopodobny poziom doświadczenia kuriera to wysokie.

Przeprowadzono testy niezależności oraz obliczono miary wzajemnej informacji (mutual information), z których wynika:

  • Istnieje silna zależność między dystansem a czasem dostawy, nawet po uwzględnieniu typu pojazdu (p < 2.2e-16).

  • Ruch drogowy warunkowany porą dnia również ma wpływ na czas dostawy (p = 0.02382).

  • Doświadczenie kuriera warunkowane ruchem drogowym wpływa istotnie na czas dostawy (p = 0.0447).

  • Nie stwierdzono istotnej zależności między dystansem a doświadczeniem kuriera (p = 0.8657) ani między typem pojazdu a czasem dostawy w warunkowaniu na Traffic + Time (p = 0.6818).

Wnioski końcowe

  • Najsilniejszym determinantem długiego czasu dostawy jest daleki dystans, co było intuicyjne.

  • Wbrew oczekiwaniom, typ pojazdu oraz doświadczenie kuriera mają ograniczony wpływ na przewidywanie czasu dostawy w kontekście innych zmiennych.

  • Sieci bayesowskie okazały się efektywnym narzędziem do modelowania zależności i wnioskowania probabilistycznego w kontekście usług dostawczych.

  • Model może być rozszerzony lub zastosowany praktycznie do oceny ryzyka opóźnień, planowania tras, alokacji kurierów czy szacowania SLA (Service-Level Agreement).
